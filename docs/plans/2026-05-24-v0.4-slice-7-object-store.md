# Slice 7 — v0.4 sesh-shim: Object Store URI translation

**Branch:** `feat/v0.4-shim-slice-7-object-store`

## 1. Goal

Make `obj://` URIs that adapters embed in outbound A2A `Part.URL` fields downloadable over plain HTTPS by remote A2A clients, without changing the adapter contract or the sesh-ops storage layout. Two surfaces land together: an outbound URL rewriter that runs at the single point where the shim serialises `messages.Message` to A2A wire JSON, and an auth-gated `GET /obj/...` download endpoint that streams the bytes from `jetstream.ObjectStore`.

## 2. Success criteria

1. Outbound A2A Messages — JSON-RPC results of `tasks/get`, `tasks/list`, `message/send`, and SSE `event: message` frames in `message/stream` / `tasks/subscribe` — with a `Part` whose `url` starts with `obj://` are rewritten to absolute HTTPS rooted at `--gateway-url`.
2. `GET <gateway-url>/obj/<scope-kind>/<scope-id>/<task-id>/<artifact-id>` with `agent.read` Bearer returns the bytes with correct `Content-Type`, `Content-Length`, `ETag`, `Content-Disposition`, `Cache-Control` headers, streamed without buffering.
3. 401 (`WWW-Authenticate: Bearer`) on missing auth; 400 on malformed path/scope; 404 on missing bucket or object; `sesh_shim_http_requests_total{route="/obj"}` increments per path-status pair.
4. When `--gateway-url` is empty, translator passes `obj://` through unchanged; logs INFO once per process.
5. Round-trip: adapter Put 2 MB blob → Part references `obj://` → shim SSE emits HTTPS URL → GET returns identical bytes.
6. Cross-stack guard: byte-equality test between `objects.URI` output and translator output.

## 3. Non-goals (deferred)

- Inbound uploads (multipart → ObjectStore → `obj://`). Adapters write directly via the SDK.
- Per-task ACL filtering on downloads. Coarse `agent.read` gate.
- Presigned/short-lived URLs.
- HTTP `Range` / partial downloads / `HEAD /obj/...`.
- Per-bucket cleanup / TTL (adapter concern).
- TS SDK realignment to the Go `obj://` form — see Risks §1.
- CI (Slice 8), E2E (Slice 9).

## 4. Ground truth verified during exploration

- `sesh-ops/objects.URI(kind, id, task, art)` emits `obj://sesh_objects_<kind>_<id>/<task>/<art>` (bucket name as authority). `ParseURI` is strict inverse.
- `sesh-ops/objects.Open(ctx, js, kind, id) → jetstream.ObjectStore`. Callers use raw store methods (Put/Get/GetInfo).
- `sesh-ops/objects.Key(taskID, artifactID) → "<task>/<art>"` — path-to-key identity.
- `internal/shim/a2a/message.go::ToWireMessage` is the SOLE outbound serialisation boundary. Both JSON-RPC handlers and SSE bridge call it.
- `internal/shim/auth.Middleware(cfg.Auth)` already used by `POST /a2a`; reuse.
- `cmd/sesh-shim/main.go` already reads `--gateway-url` and threads into `card.NewComposer`. Plumb same into `server.Config`.
- TS SDK `parseObjectURL` uses a DIFFERENT shape (`obj://<kind>/<id>/<task>/<art>`). Out-of-scope cross-stack divergence — Risk §1.

## 5. File-by-file design

### 5.1 `internal/shim/a2a/translator.go` (NEW)

```go
type Translator struct {
    GatewayURL string  // empty ⇒ pass-through
    Log        *slog.Logger
    warnOnce   sync.Once
}

func NewTranslator(gatewayURL string, log *slog.Logger) *Translator
func (t *Translator) ToWireMessage(m *messages.Message) (json.RawMessage, error)
```

`ToWireMessage` defensively copies Parts before mutation. Per-Part: skip non-`obj://`; if `GatewayURL == ""` log INFO once and pass through; else parse via `objects.ParseURI` (skip + WARN on malformed); rewrite to `<gateway>/obj/<kind>/<id>/<task>/<art>` with `url.PathEscape` on each segment.

## 7. Decisions resolved

| # | Decision | Resolution |
|---|---|---|
| D1 | GatewayURL plumbing | (c) `a2a.Translator{GatewayURL,Log}` struct constructed once at boot. Free-function `ToWireMessage` retained as deprecated shim for Slice 1 continuity. |
| D2 | Path scheme | (a) `/obj/<scope-kind>/<scope-id>/<task-id>/<artifact-id>` — explicit, mirrors `obj://` segments, no bucket-name leak. |
| D3 | Auth scope | Reuse `agent.read` (same as `ListTasks`). Per-task ACL deferred. |
| D4 | Streaming | `io.Copy(w, obj)` after `GetInfo`-sourced headers. No buffering. |
| D5 | No GatewayURL | Pass-through with `sync.Once`-gated INFO log. Pre-v0.4 native clients keep working. |
| D6 | Cache headers | `Cache-Control: public, max-age=31536000, immutable` (artifact IDs are content-addressed). |
| D7 | Content-Length + ETag | Set from `GetInfo`; ETag is digest hex (strong validator). HEAD deferred. |

## 8. Open questions

1. **TS SDK URI form divergence**. `sesh-channels/sdk/src/objects.ts::parseObjectURL` accepts `obj://<kind>/<id>/<task>/<art>` (4-segment split); Go `sesh-ops/objects/uri.go::URI` emits `obj://sesh_objects_<kind>_<id>/<task>/<art>` (glued). Slice 7 is correct in isolation (Go ↔ Go), but TS-SDK-built adapters that mint their own `obj://` URLs would produce strings this translator can't parse. SLICE 7 SCOPE: flag in PR + surface to orch as a NEW cross-stack contract bug separate from #121/#122/#124. Defensive parser for TS form deferred — strict alignment work.
2. **`Content-Type` header presence**. Go SDK `objects.putObject` doesn't yet propagate `MediaType`/`Filename` from caller's `Part` into ObjectMeta. Until SDK grows that, downloads default to `application/octet-stream`. Accept default; `Content-Disposition: attachment` makes browsers download cleanly.
3. **Metrics label cardinality for `/obj`**. Route label `/obj` constant; can't distinguish per-scope traffic. Acceptable for v0.4 (matches `/a2a` aggregation).

## 9. Risks

1. **TS SDK divergence** (Open Q §1). If a TS-SDK adapter ships before alignment, its `obj://` URLs pass through unchanged (`objects.ParseURI` rejects them); clients receive unreachable `obj://` URLs. Mitigation: WARN log on parse failure for ops visibility.
2. **`scope.Bucket` validation**. `workflow` scope-ids must be 8 or 32 hex; mapped to 400 but terse. Acceptable for v0.4.
3. **Adapter Put / shim Get race**. Multi-replica JetStream could see transient 404 between Put and visible Get. Acceptable for v0.4 (single-replica dev cluster).
4. **`io.Copy` partial write post-headers**. Client disconnect mid-stream: error after headers sent. Log WARN; can't update HTTP status. Standard streaming-server tradeoff.
5. **Translator refactor blast radius**. Touches 3 packages (a2a, sse, methods). Mitigated by deprecated free-function shim preserving Slice 1 contract.
