# v0.4 sesh-shim Slice 3 — SendMessage + SendStreamingMessage + SubscribeToTask + idempotency

**Branch:** `feat/v0.4-shim-slice-3-send-stream-subscribe`

## 1. Goal

Slice 3 makes the shim *useful*: any stock A2A v1.0 client can post a message to a sesh adapter, stream the adapter's reply via SSE, and reattach to an in-flight task. It wires three new JSON-RPC methods (`SendMessage`, `SendStreamingMessage`, `SubscribeToTask`) plus the existing `GetTask`/`GetExtendedAgentCard` from Slices 1+2 into one cohesive request/response/stream surface, driven through `sesh-ops/messages` + `sesh-ops/artifacts` and `internal/subject`'s `PromptV2` builder. It also nails down the load-bearing cross-stack design point — A2A `ROLE_USER`/`ROLE_AGENT` ↔ sesh-ops `"user"`/`"agent"` — in a single boundary package, and implements the spec-mandated `messageId` idempotency contract.

## 2. Success criteria

Verifiable locally:

- `SendMessage` with no `params.message.taskId` creates a Task in `sesh_tasks_<scope>_<id>` and appends the first message to `sesh_messages_<scope>_<id>` at `<taskID>.<msgID>`. Returns the raw Task JSON.
- `SendMessage` with `params.message.taskId` set appends to the existing task. Returns the updated Task JSON.
- `SendMessage` publishes a v2 prompt to `agents.prompt.v2.<machine>.<project>.<session>.<role>[.<inst>]` after the KV append (built via `subject.PromptV2`). Adapter wakes via subscription on the queue group `PromptV2QueueGroup`.
- `SendMessage` retried with the SAME `messageId` and byte-identical body returns the stored record as success (no error, no double-write).
- `SendMessage` retried with the SAME `messageId` and DIFFERENT body returns JSON-RPC `-32602` with `data.kind == "messageId-conflict"`.
- A2A `Message.Role: "ROLE_USER"` on the wire → stored as `"user"` in sesh-ops; sesh-ops `"agent"` in storage → serialized back as `"ROLE_AGENT"` on every outbound A2A path.
- `SendStreamingMessage` returns `Content-Type: text/event-stream`, appends + publishes (like `SendMessage`), then emits one SSE event per `messages.WatchEvent` (named `message`) and one per `artifacts.Update` (named `artifact-update`) until client disconnect or task terminal-state. Emits `:keepalive\n\n` comment every 25s.
- `SubscribeToTask` returns the same SSE shape with `WatchOpts.IncludeHistory: true`, NO inbound append, NO publish.
- Client disconnect (`r.Context().Done()`) stops both watchers within 500ms; no goroutine leak (race-detector + manual assertion).
- All Slice 1 + Slice 2 tests still green; no regressions.
- `go test ./... -race -count=1` passes.
- `gofmt -l ./internal/shim ./internal/subject ./cmd/sesh-shim` clean.

### 4.5 `internal/shim/sse/bridge.go` (NEW)

Single tight helper. `~120 LOC` total including keepalive.

```go
package sse

import (
    "context"
    "encoding/json"
    "fmt"
    "net/http"
    "time"

    "github.com/danmestas/sesh-ops/artifacts"
    "github.com/danmestas/sesh-ops/messages"
    "github.com/danmestas/sesh/internal/shim/a2a"
)

type Options struct {
    KeepaliveInterval time.Duration // default 25s
}

// Bridge writes SSE events to w as messages and artifacts arrive on
// the supplied watch channels. Returns when ctx is cancelled, both
// watch channels close, or writing to w fails.
//
// Emits ":keepalive\n\n" comment every KeepaliveInterval (default 25s)
// so reverse proxies don't kill idle adapter streams (cloudflare's
// 100s default; nginx default 60s; etc.).
func Bridge(
    ctx context.Context,
    w http.ResponseWriter,
    msgCh <-chan messages.WatchEvent,
    msgStop func(),
    artCh <-chan artifacts.Update,
    opts Options,
) error
```

Implementation: select loop over `ctx.Done()`, `msgCh`, `artCh`, `time.NewTicker(opts.KeepaliveInterval)`. Nil-channel idiom (set channel to nil after close) lets the select drop closed channels without hot-spinning. Each event emits `event: <name>\ndata: <json>\n\n`. Keepalive emits `:keepalive\n\n` (SSE comment per spec — clients ignore it but proxies see traffic).

`defer msgStop()` for cleanup. Artifacts watcher lifecycle is bound to `r.Context()` (sesh-ops contract), so canceling `ctx` cleans both.

### 4.10 `internal/shim/methods/send.go` (NEW)

`sendMessage` handler. Steps:

1. Decode params: `SendMessageParams{Message json.RawMessage}` (A2A spec; ignores `configuration`).
2. `a2a.FromWireMessage(params.Message)` → `*messages.Message`.
3. If `m.TaskID == ""`: generate ULID taskID + contextID; write A2A-shaped Task JSON via `kv.Create(ctx, taskID, taskBytes)` to `sesh_tasks_*` (NOT via `sesh-ops/task.Add` — see Slice 2 D-task). Set `m.TaskID` / `m.ContextID`.
4. If `m.TaskID != ""`: confirm task exists (`kv.Get(ctx, m.TaskID)`). Missing → `jsonrpc.ErrTaskNotFound`.
5. If `m.ID == ""`: generate ULID.
6. Open messages KV: `scope.Bucket(scopeKind, scopeID, "messages")`. Defensively `js.CreateKeyValue` (idempotent on existing).
7. `messages.Append(ctx, kv, messages.AppendOpts{Message: m})`:
   - `(stored, nil)`: success.
   - `(nil, messages.ErrMessageIDConflict)`: return `jsonrpc.ErrInvalidParams.WithData({kind: "messageId-conflict"})`.
   - `(nil, otherErr)`: log + `jsonrpc.ErrInternal`.
8. **Publish v2 prompt** (resolved D-publish):
   ```go
   subj, err := subject.PromptV2(subject.Coord{
       Machine: deps.Machine, Project: deps.ScopeID, // session-scoped
       Session: deps.ScopeID, Role: deps.AgentKey.Agent,
   })
   if err == nil {
       _ = deps.NC.Publish(subj, params.Message) // fire-and-forget; adapter wakes via queue-group
   }
   ```
   Failure is logged but does not fail the JSON-RPC call — KV state is the source of truth; the publish is a wake-up signal that adapters can also detect via their KV watch.
9. Re-read Task bytes, return as `json.RawMessage`.

### 4.12 `internal/shim/methods/stream.go` (NEW)

`SendStreamingMessage` handler — takes `(w, r, params)`. Steps:

1. Same steps 1-8 as `sendMessage` (decode, validate, generate IDs, append, publish).
2. Pre-error path: if any of those fail BEFORE SSE headers are written, emit plain JSON-RPC error (no SSE).
3. Defensively `js.CreateKeyValue` for `sesh_artifacts_<scope>_<id>`.
4. Open watchers:
   ```go
   msgCh, msgStop, _ := messages.Watch(r.Context(), msgsKV, taskID, messages.WatchOpts{IncludeHistory: false})
   artCh, _ := artifacts.Watch(r.Context(), artsKV, taskID, artifacts.WatchOpts{IncludeHistory: false})
   ```
   `IncludeHistory: false` — we just appended; tail-only avoids echo.
5. `sse.Bridge(r.Context(), w, msgCh, msgStop, artCh, sse.Options{KeepaliveInterval: 25 * time.Second})`.

## 8. Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| A2A `MessageRole` wire-shape drift | Low (a2a-go pinned v2.3.1) | Translation isolated in `internal/shim/a2a/role.go` |
| SSE backpressure — slow client backs up unbuffered watcher | Medium | Bridge honors `ctx.Done()` on every send; stuck client gets dropped at next cancel |
| Watcher leak on client disconnect | High if mishandled / Low with design | `defer msgStop()` + tests assert via goroutine-count delta |
| Idempotency body-comparison edge case (nil-vs-empty metadata) | Medium | sesh-ops `messages.equalForIdempotency` handles this; add test for `metadata: null` vs `metadata: {}` vs omitted |
| Bucket-not-found on first-ever Append | Medium | Defensively `js.CreateKeyValue` (idempotent) |
| `methods/` promotion breaks Slice 1/2 tests | Low | Tests run through same JSON-RPC wire surface; only internal routing changes |
| Publish race — adapter wakes before KV write commits | Low (CAS on Append precedes Publish) | Adapters already idempotent on duplicate KV reads |
| Proxy idle-timeout kills SSE | Was: high → with keepalive: very low | `:keepalive` every 25s beats every common proxy default |

## Phase 3 Ousterhout review (applied)

**`internal/shim/a2a/` translation layer** — shallow (~25 LOC for role.go), but the cross-stack wire-shape divergence is the single most fragile contract in this slice. A named package gives a grep-able home + a test file that pins the mapping. Earns its keep. Worth the shallowness.

**`sse.Bridge`** — owns SSE wire format + flusher + two-channel select + ctx-cancellation + role-translation hook + stop-func cleanup + keepalive. ~120 LOC of real behavior. Not a pass-through. Earns its keep.

**`methods/` subpackage promotion** — 4 methods this slice, 6 by Slice 4. Stdlib convention threshold (4-5 related symbols across files) clears in Slice 3 and keeps clearing. Promoting now is the cheaper sequence than promoting under harder Slice-4 conditions. Confirmed: not premature.

**Over-generalization watch.** `Deps` struct on dispatcher stays tight (NC, JetStream, ScopeKind, ScopeID, AgentKey, Machine, Log). Anything else gets added in the slice that first needs it. Dispatcher is concrete, not an interface — no `MessageHandler` premature abstraction.

**Phase 3 cuts applied to the plan-agent's draft:**

1. **`message.go` Part-flattening claim was wrong.** Plan-agent said A2A `Part.Content` is a tagged union to map into sesh-ops's flat Part. Verified against `a2a-go/v2/a2a/core.go:582-756` and sesh-ops `messages/messages.go` — both sides already use FLAT Part shape (no `Content` wrapper, no `kind` discriminator). Scope trimmed to role-only translation; Parts copy through. Saves a wrong implementation.

2. **D-publish moved from open question → resolved YES.** Plan-agent left "should SendMessage publish to `agents.prompt.v2.*`?" as an open question with a defer. But: subjects.ts defines `PROMPT_V2_QUEUE_GROUP` (`"agents-prompt-v2"`), Slice 2's whole subject-builder package exists for exactly this publish, and adapters subscribe to the queue group for load-balanced delivery. The KV-watch pattern is for the TASK lifecycle, not direct prompts. Shipping Slice 3 without the publish would mean shipping a non-functional adapter wake path. Locked YES.

3. **D-keepalive added.** Plan-agent deferred SSE heartbeat to Slice 4. But proxy idle-timeouts (nginx 60s default, cloudflare 100s) will silently kill in-flight adapter streams. ~5 LOC in `sse.Bridge` is the right cost-benefit. Shipping in Slice 3.

4. **External dep count.** Adds `oklog/ulid/v2` direct (likely already transitive via sesh-ops). Acceptable.

No structural rework required beyond the three cuts above. Proceed to Phase 4.
