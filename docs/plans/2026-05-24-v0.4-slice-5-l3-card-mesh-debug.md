# Slice 5 тАФ v0.4 sesh-shim: L3 AgentCard + extended card + `sesh mesh card` debug

**Branch:** `feat/v0.4-shim-slice-5-l3-card`

## 1. Goal

Add the L3 contribution layer to the shim's `AgentCard` composition pipeline, ship a real (non-stub) `GetExtendedAgentCard` JSON-RPC handler that fetches the auth-gated extended card from the adapter via `agents.card.extended.*`, and ship a `sesh mesh card <agent>` debug subcommand that exercises both subjects from the operator CLI. After Slice 5 the shim covers the full v0.4 AgentCard surface (L1+L2+L3 public, L1+L2+L3 extended), end-to-end byte-compatible with the sesh-channels SDK's `registerAgentCard`/`registerAgentCardExtended` wire shape.

## 2. Success criteria

- `Composer.Compose` issues exactly one `nats.Request` to `agents.card.get.<agent>.<owner>.<name>` per cache miss, bounded by `queryWindow` (default 500ms). On reply: decode JSON as `cardPartial`, overlay onto L1+L2 (D1: adapter wins per-field for the L3-owned fields).
- `Composer.Compose` returns L1+L2 only when the L3 fetch times out or yields a non-JSON reply; logs INFO with `key=` and reason. NEVER errors due to L3 absence (D2).
- Cached signed bytes (5min TTL) reflect L1+L2+L3; cache key unchanged (`AgentKey{Agent, Owner, Name}`).
- `AgentKey` gains a `Name` field (currently missing). `cmd/sesh-shim/main.go` gains `--name` CLI flag defaulting to `cli.Agent`. All `AgentKey{Agent:..., Owner:...}` literals in tests + main are updated.
- `GetExtendedAgentCard` returns a signed extended card when (a) the principal carries scope `agent.read.extended` (D3), AND (b) the adapter replies to `agents.card.extended.<agent>.<owner>.<name>` within `queryWindow`. Otherwise: `jsonrpc.ErrExtendedCardNotConfigured` (`-32007`).
- Extended card is composed per-request (not cached тАФ D4): re-compose L1+L2 base via `Composer.ComposeBase`, overlay extended `cardPartial`, sign via `Signer.SignCard`, return as `json.RawMessage`.
- `auth.NoopValidator` dev-scope set includes `agent.read.extended` so `--auth=none-dev-only` mode can exercise the new path end-to-end.
- `sesh mesh card <agent> [--owner=тАж --name=тАж --extended --window=500ms -o json|tree]` connects via the same NATS-URL-resolution path as `mesh list`, issues `nats.Request` to the chosen subject, decodes the reply as `cardPartial`, renders, and exits 0 on success / 1 on timeout-or-no-reply.
- `go test ./... -race -count=1` passes; `go vet ./...` and `gofmt -l ./...` clean.
- Cross-stack golden: a new test decodes a hand-authored JSON document matching the TS `AgentCardPartial` byte-shape and confirms field-by-field; comment points at `sesh-channels/sdk/src/card.ts`.

- L3 cache invalidation when adapter departs (Slice 6+).

const extendedReadScope = "agent.read.extended"

func (d *Dispatcher) getExtendedAgentCard(ctx, _) (any, *jsonrpc.Error) {
    p, ok := auth.FromContext(ctx)
    if !ok || !hasScope(p, extendedReadScope) {
        return nil, jsonrpc.ErrExtendedCardNotConfigured
    }
    partial, found := d.deps.Composer.FetchExtended(ctx, d.deps.AgentKey)
    if !found { return nil, jsonrpc.ErrExtendedCardNotConfigured }
    base, err := d.deps.Composer.ComposeBase(ctx, d.deps.AgentKey)
    if err != nil { /* log + ErrInternal */ }
    d.deps.Composer.ApplyPartial(base, partial)
    signed, err := d.deps.Signer.SignCard(base)
    if err != nil { /* log + ErrInternal */ }
    return json.RawMessage(signed), nil
}
