# Slice 6 — v0.4 sesh-shim: A2A push-notification CRUD + durable delivery worker

**Branch:** `feat/v0.4-shim-slice-6-push-notifications`

## 1. Goal

Add the 4 A2A `*TaskPushNotificationConfig` JSON-RPC methods to the shim, plus a durable JetStream-backed delivery worker that POSTs `TaskStatusUpdateEvent` payloads to registered webhooks with exponential backoff, SSRF defense, and shim-held AES-256-GCM credential encryption. Failed deliveries land in the `sesh_notifyfail_*` bucket via `notifications.AppendFail` for forensic inspection. After Slice 6, the shim covers the A2A push-notification surface end-to-end.

## 2. Success criteria

- 4 new JSON-RPC methods on `/a2a`:
  - `CreateTaskPushNotificationConfig` (scope: `agent.notify.write`) → encrypts `Auth.Credentials` to `enc:<base64-gcm>` → `notifications.Set` → returns the stored `NotifyConfig` with plaintext re-attached for the response.
  - `GetTaskPushNotificationConfig` (scope: `agent.notify.read`) → `notifications.Get` → decrypts → returns `NotifyConfig`.
  - `ListTaskPushNotificationConfigs` (scope: `agent.notify.read`) → `notifications.List` → decrypts each → returns `{configs: [...]}`. Missing scope returns `{configs: []}` HTTP 200 (mirrors `ListTasks`).
  - `DeleteTaskPushNotificationConfig` (scope: `agent.notify.write`) → `notifications.Delete` → returns `{}`. Idempotent.
- Encryption envelope: `EncryptCredentials(plaintext) → "enc:" + base64(nonce||ciphertext||tag)` via AES-256-GCM with a 32-byte shim-held key. `DecryptCredentials` tolerates pre-Slice-6 raw values (`ErrLegacyPlaintext` sentinel + WARN log).
- SSRF guard rejects: non-`https://` (unless `--dev` + host ∈ `{localhost,127.0.0.1,::1}`); hostnames resolving to RFC 1918, link-local, loopback, ULA, unspecified. DNS rebinding defense: re-resolve at both Create-time AND each delivery.
- Durable JS consumer `shim-push-worker-<scope-kind>-<scope-id>` (single per scope) subscribes to Tasks KV stream via `tasksKV.WatchAll(ctx, jetstream.IgnoreDeletes(), jetstream.UpdatesOnly())`. On each Put: parse status.state → `notifications.List(taskID)` → fan-out per config: SSRF re-check → decrypt → POST `TaskStatusUpdateEvent` with `Authorization: <Scheme> <plaintext>`, 10s HTTP timeout.
- Retry policy: 5 attempts at 1s/4s/16s/64s delays (override via `--push-max-retries`; var swappable for tests). Final failure → `notifications.AppendFail(taskID, configID, attempt, errStr)`.
- Worker lifecycle: started by `server.Run` when `--push-worker-disabled=false` AND key loaded; stopped on ctx cancel; in-flight bounded by ShutdownGrace.
- 3 new CLI flags: `--push-encryption-key`/`SESH_SHIM_PUSH_ENCRYPTION_KEY` (hex env wins, file fallback, dev ephemeral); `--push-worker-disabled`; `--push-max-retries`.
- NoopValidator dev-scopes extended with `agent.notify.{read,write}`.
- `go test ./... -race -count=1` passes; `go vet`/`gofmt -l` clean.
- Plaintext credentials NEVER appear in KV values, log records, error responses, dead-letter entries. Verified by `TestPush_CredentialsNeverLeak_*`.

## 3. Non-goals (deferred)

- Key rotation hot-swap — restart required for v0.4.
- Dead-letter inspection API endpoint — ops `nats kv get sesh_notifyfail_*` for v0.4.
- Push on non-status events (artifacts, messages).
- Webhook content-type negotiation — always `application/json`.
- Push for non-shim-scope tasks. One worker per shim scope.
- Object Store (Slice 7), CI (Slice 8), E2E (Slice 9).
- HTTP connection-pool tuning beyond `MaxIdleConnsPerHost=10` and 10s timeout.
- Circuit-breaker reading notifyfail.

### 5.3 `internal/shim/push/ssrf.go` (NEW)

```go
type Resolver func(host string) ([]net.IP, error)
```

Default Resolver wraps `net.DefaultResolver.LookupIPAddr`. `isBlockedIP`: `IsLoopback || IsLinkLocalUnicast || IsLinkLocalMulticast || IsUnspecified || IsMulticast || IsPrivate` (Go 1.17+ covers RFC1918 + ULA). HTTP allowed only when `devAllowLocalhost && host∈{localhost,127.0.0.1,::1,[::1]}`.

### 5.4 `internal/shim/push/ssrf_test.go` (NEW)

~15 table cases: https public/private, http localhost dev/prod, IPv4/IPv6 loopback/link-local/ULA/RFC1918, 169.254.169.254 (cloud metadata), 0.0.0.0, ftp://, empty/malformed URLs, DNS-rebinding stub (resolver returns public→private).

### 5.5 `internal/shim/methods/pushconfig.go` (NEW)

Four handlers. Key invariants:
- Decode `params.pushNotificationConfig` directly into `notifications.NotifyConfig` (wire-equivalent to A2A `PushConfig`).
- Encrypt at boundary in `create`: copy `cfg.Auth`, set `Credentials = enc`, persist. Response returns ORIGINAL cfg (plaintext echoed).
- `create` runs SSRF guard at registration; verifies task exists (`ErrTaskNotFound` if missing); generates ULID for blank ID.
- `get`/`list` decrypt at boundary; tolerate `ErrLegacyPlaintext` with WARN.
- `list` returns `{configs:[]}` HTTP 200 on missing read scope (mirrors `ListTasks`).
- `delete` idempotent on NotFound.
- All 4 return `ErrPushNotConfigured` when `PushKey == nil`.
- Reuse `hasScope` (list.go) + `openOrCreateKV` (send.go).

### 5.6 `internal/shim/methods/pushconfig_test.go` (NEW)

~17 tests: Create happy-path-encrypted (raw KV bytes have `enc:` prefix, plaintext NEVER present), Create assigns blank ID, Create task-not-found, Create rejects SSRF table, Create missing scope, Create missing PushKey; Get decrypts, Get not-found, Get missing scope, Get legacy plaintext warns+returns-as-is; List decrypts all, List missing-scope returns empty, List empty bucket; Delete happy, Delete idempotent, Delete missing scope; AllMethods no-plaintext-in-logs (slog buffer scan).

### 5.7 `internal/shim/push/worker.go` (NEW)

```go
type WorkerConfig struct{ NC; JS; ScopeKind; ScopeID; PushKey; HTTPClient; Log; MaxRetries }
type Worker struct{ cfg WorkerConfig; wg sync.WaitGroup }

func NewWorker(cfg WorkerConfig) *Worker
func (w *Worker) Run(ctx context.Context) error
func (w *Worker) Wait()

var RetrySchedule = []time.Duration{1*time.Second, 4*time.Second, 16*time.Second, 64*time.Second}
```

`Run` opens tasks/notifycfg/notifyfail KVs, then `tasksKV.WatchAll(ctx, IgnoreDeletes(), UpdatesOnly())`. Per update: `parseTaskState` → non-empty → goroutine `deliverAll` → `notifications.List` → fan-out `deliverOne` per config. `deliverOne` loops `1 + len(RetrySchedule)` times: SSRF re-check (hard-fail), `post`, on non-2xx `AppendFail` + sleep next backoff. `post`: builds request, sets Content-Type, optional `X-A2A-Notification-Token`, `Authorization: <Scheme> <decrypted-plaintext>`. `buildStatusEvent` produces `{kind:"status-update", taskId, status, finalState}`. HTTP client: `Timeout: 10s`, `MaxIdleConnsPerHost: 10`.

## 11. Risks

- Key rotation requires restart. Slice 9 dual-key support deferred.
- Dead-letter bucket grows unbounded. Mitigation: `KeyValueConfig{TTL: 168*time.Hour}` at create.
- Time-flaked worker tests from real backoff. Mitigation: RetrySchedule swap in `t.Cleanup` (§6.5).
- Worker downtime = no deliveries. Single shared consumer; HA deferred.
- DNS resolver flakes → spurious SSRF failures. Stdlib resolver caches via OS.
- `UpdatesOnly()` skip-historical. Document.
- Envelope versioning. `"enc:"` no version byte; extend with `"enc2:"` later if needed.

## Phase 3 Ousterhout review (applied)

Plan-agent's pre-review is solid:

- **`push` subpackage justified**: worker has its own lifecycle + retry state; crypto + ssrf reused by both CRUD methods AND worker. Putting in methods/ would force worker (lives elsewhere) to import methods/ → cycle. Real boundary, not naming.
- **Hard-coded retry policy with one knob**: `--push-max-retries` (count only); per-step delay tuning would be speculative interface bloat. Matches `casRetryBackoffs` precedent from Slice 4.
- **`enc:` 4-byte prefix justified**: smallest forward-compat marker. Distinguishes new code path from legacy plaintext. sesh-ops package doc mentions `enc:<base64>` as expected wrapper.

External deps: 0 new. AES-GCM via stdlib. Net.IP classification via stdlib (`ip.IsPrivate()` covers RFC1918 + ULA since Go 1.17). No retry library.

`PushKey` on `methods.Deps` is real new state but Step 4 isolates it: pure config plumbing, zero behavior change.

Proceed to Phase 4.
