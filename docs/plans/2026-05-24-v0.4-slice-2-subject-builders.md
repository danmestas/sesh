# v0.4 sesh-shim Slice 2 — Subject builders + sesh-ops bucket derivation

**Branch:** `feat/v0.4-shim-slice-2-subjects`

## 1. Goal

Introduce a Go subject-builder package (`internal/subject/`) that mirrors `sesh-channels/sdk/src/subjects.ts` byte-for-byte, and switch the shim's existing JetStream KV bucket-name derivation from an inline `fmt.Sprintf` to `github.com/danmestas/sesh-ops/scope.Bucket(...)`. Unblocks every Slice 3+ method (`SendMessage`, `SendStreamingMessage`, `SubscribeToTask`, L3 `CardGet`/`CardExtended` in Slice 5) by giving them a single lint-friendly source of truth for the on-the-wire subject strings, and gets the shim onto the sanctioned bucket-derivation function (sanitization, scope-kind validation) without changing the existing zero-translation Task read semantics. Deferred to follow-up upstream sesh-ops issues: adopting `conn.Connect` (would regress shim client naming and reconnect policy) and adopting `task.Get` (would lose A2A-specific Task fields).

### 4.1 `internal/subject/subjects.go` (NEW)

Mirrors `/Users/dmestas/projects/sesh-channels/sdk/src/subjects.ts` byte-for-byte.

Exported symbols:

```go
package subject

// PromptV2QueueGroup is the NATS micro queue group for v2 prompt
// handlers across replicas of the same role. MUST match the value
// used on the publisher side. Mirrors PROMPT_V2_QUEUE_GROUP in
// sesh-channels/sdk/src/subjects.ts.
const PromptV2QueueGroup = "agents-prompt-v2"

// Coord identifies a v2 prompt destination. Mirrors the TS `Coord`
// interface. Inst is optional (empty string means "no inst token").
type Coord struct {
    Machine string
    Project string
    Session string
    Role    string
    Inst    string
}

// InvalidTokenError reports a subject token that violates the
// lowercase-dot-separated convention (no '.', '*', '>', whitespace,
// non-empty). Mirrors InvalidSubjectTokenError in subjects.ts.
type InvalidTokenError struct {
    Token  string
    Reason string
}

func (e *InvalidTokenError) Error() string

// PromptV2 returns agents.prompt.v2.<machine>.<project>.<session>.<role>[.<inst>].
// Mirrors promptV2(c) in subjects.ts.
func PromptV2(c Coord) (string, error)

// ParsePromptV2 reverses PromptV2. Each extracted token is re-validated
// via validateToken so the parser is no more lenient than the builder.
// Returns an error if subject doesn't match the agents.prompt.v2.* shape
// or any token fails validation.
func ParsePromptV2(subject string) (Coord, error)

// TaskStream returns agents.task.stream.<scopeKind>.<scopeId>.<taskId>.
// Mirrors stream({scopeKind, scopeId, taskId}) in subjects.ts.
func TaskStream(scopeKind, scopeID, taskID string) (string, error)

// CardGet returns agents.card.get.<agent>.<owner>.<name>.
// Mirrors card({agent, owner, name}) in subjects.ts.
func CardGet(agent, owner, name string) (string, error)

// CardExtended returns agents.card.extended.<agent>.<owner>.<name>.
// Mirrors cardExtended({agent, owner, name}) in subjects.ts.
func CardExtended(agent, owner, name string) (string, error)

// validateToken rejects empty tokens or tokens containing '.', '*',
// '>', or any whitespace. Package-internal — the TS side exports it
// because TS lacks package-internal symbols; Go doesn't need to.
func validateToken(token string) error
```

What it does:
- All public builders call `validateToken` on each token and return `*InvalidTokenError` on failure.
- `PromptV2` returns `"agents.prompt.v2." + strings.Join([]string{m, p, s, r}, ".")` plus optional `"." + inst`.
- `ParsePromptV2` splits on `.`, asserts prefix `agents.prompt.v2.`, returns the remaining 4 or 5 tokens as a `Coord`. Inst empty when only 4 follow the prefix. Each extracted token is re-validated via `validateToken` (symmetry: parser no more lenient than builder).
- `TaskStream`, `CardGet`, `CardExtended` are straight `fmt.Sprintf` after validation.
- `validateToken`: empty → reason `"empty"`; contains `.`, `*`, `>`, or any rune satisfying `unicode.IsSpace` → reason `"contains reserved character (. whitespace * >)"` (text matches TS exactly so error grep is consistent cross-stack).

What it does NOT do:
- Does NOT escape or percent-encode invalid tokens (callers sanitize upstream — same contract as TS).
- Does NOT validate the scope-kind enum (`{hub, project, session, workflow, agent}`); `TaskStream` only checks token-shape. Enforcement lives in `scope.Bucket` already.
- Does NOT include `PromptV1` (legacy v0.3 shape not in v0.4 mirror).
- Does NOT include `MustPromptV2` (tests use error-returning form).

Each function carries a `// Mirrors X in subjects.ts` line so future drift is grep-able.

### 4.2 `internal/subject/subjects_test.go` (NEW)

Table-driven, stdlib `testing` only (no `testify`).

Test groups:

1. **`TestPromptV2_Build`** — literal-string assertions:
   - `{m1,p1,s1,r1,i1}` → `"agents.prompt.v2.m1.p1.s1.r1.i1"`
   - `{m1,p1,s1,r1,""}` → `"agents.prompt.v2.m1.p1.s1.r1"`
   - `{laptop,sesh-channels,main,worker,""}` → `"agents.prompt.v2.laptop.sesh-channels.main.worker"`
   - `{laptop,p,s,r,"1"}` → `"agents.prompt.v2.laptop.p.s.r.1"`

2. **`TestPromptV2_RejectsBadTokens`** — table of bad coords returning `*InvalidTokenError`:
   - `{m.1, p, s, r, ""}` — `.` in token
   - `{m, *, s, r, ""}` — `*`
   - `{m, p, s, >, ""}` — `>`
   - `{m, p, s, "r ", ""}` — whitespace
   - `{"", p, s, r, ""}` — empty machine
   - Positive: `{m, p, s, r, ""}` — empty Inst is omitted, NOT validated as a token (one intentional Go/TS divergence: TS distinguishes `undefined` from `""`; Go uses `""` to mean "omit").

3. **`TestPromptV2_RoundTrip`** — `c2, _ := ParsePromptV2(s); require c == c2` for canonical Coords (with and without `Inst`).

4. **`TestParsePromptV2_Rejects`** — bad inputs:
   - `""`, `"foo.bar"`, `"agents.prompt.v1.a.b.c.d"`, `"agents.prompt.v2.a.b.c"` (only 3 tokens), `"agents.prompt.v2.a.b.c.d.e.f"` (6 tokens).
   - `"agents.prompt.v2.a..s.r"` (empty token within split) → must error per the symmetry rule.

5. **`TestTaskStream`** — `("session", "abc123", "01HX")` → `"agents.task.stream.session.abc123.01HX"`.

6. **`TestCard_Builders`**:
   - `CardGet("claude","dmestas","main")` → `"agents.card.get.claude.dmestas.main"`
   - `CardExtended("claude","dmestas","main")` → `"agents.card.extended.claude.dmestas.main"`
   - `CardGet("cl aude", ...)` → error.

7. **`TestValidateToken`** — table of negatives `[".", " ", "\t", "\n", "*", ">", "a.b", "x y", ""]` and positives `["m1", "claude", "01HX5K0", "dmestas-main"]`.

8. **`TestPromptV2QueueGroup`** — `PromptV2QueueGroup == "agents-prompt-v2"`.

### 4.3 `internal/shim/server/methods.go` (MODIFIED)

Single change in `getTask`: replace the inline `fmt.Sprintf` with `scope.Bucket`.

```go
import (
    // ...existing...
    "github.com/danmestas/sesh-ops/scope"
)

func (s *server) getTask(ctx context.Context, params json.RawMessage) (any, *jsonrpc.Error) {
    if s.cfg.JetStream == nil {
        return nil, jsonrpc.ErrInternal.WithData(map[string]string{"reason": "JetStream not configured"})
    }
    var p getTaskParams
    if len(params) > 0 {
        if err := json.Unmarshal(params, &p); err != nil {
            return nil, jsonrpc.ErrInvalidParams.WithData(map[string]string{"err": err.Error()})
        }
    }
    if p.ID == "" {
        return nil, jsonrpc.ErrInvalidParams.WithData(map[string]string{"err": "id is required"})
    }
    bucket, err := scope.Bucket(s.cfg.ScopeKind, s.cfg.ScopeID, "tasks")
    if err != nil {
        s.log.Error("getTask: bucket derive", "scope_kind", s.cfg.ScopeKind, "scope_id", s.cfg.ScopeID, "err", err)
        return nil, jsonrpc.ErrInternal.WithData(map[string]string{"reason": "invalid scope"})
    }
    kv, err := s.cfg.JetStream.KeyValue(bucket)
    // ...rest unchanged (KV lookup, error mapping to TaskNotFoundError, return json.RawMessage(entry.Value()))...
}
```

What stays the same (intentional):
- Direct `js.KeyValue(bucket)` — does NOT route through `sesh-ops/task.Get`. Rationale: `task.Get` returns `*task.Task` (sesh-internal: `Title`, `Description`, `Status`, `Puller`, `BlockedReason`, …). The bytes stored in `sesh_tasks_*` for A2A consumers are A2A `Task` JSON (`id`, `kind:"task"`, `status:{state:"submitted"|…}`, `contextId`, `history`, `artifacts`). Decoding to sesh-internal `*task.Task` and re-marshaling would drop every A2A-specific field. Filed-as upstream issue (D-task).
- `entry.Value()` → `json.RawMessage` return shape unchanged.
- `nats.ErrBucketNotFound` and `nats.ErrKeyNotFound` both still map to `jsonrpc.ErrTaskNotFound`.
- `scope.Bucket` error → `jsonrpc.ErrInternal`.

Verify behavior parity for existing tests:
- Slice 1 seeds `Bucket: "sesh_tasks_project_abc123"`. `scope.Bucket("project", "abc123", "tasks")` → `"sesh_tasks_project_abc123"`. Identical.

What does NOT change:
- `dispatch(...)` — no new method routes.
- `getExtendedAgentCard` — still returns `ErrExtendedCardNotConfigured`.

## Phase 3 Ousterhout review (applied)

The `internal/subject` package is shallow by Ousterhout's definition — a thin layer of pure builder functions with little internal logic per function. But its deep value is *centralizing the protocol spec*: every call-site that would hand-roll `"agents.prompt.v2." + ...` now goes through a single point that (a) gets the validation contract right, (b) gets the byte-for-byte mirror right with the TS side, and (c) is the one place to grep when the protocol moves. The TS file earns its keep the same way. Shallow modules are usually a smell; this one isn't.

No pass-through wrappers introduced. `methods.go` does not gain any function-only-calling-another-function layer; the change is a single in-place expression swap.

Over-generalization watch: `Coord` struct (PromptV2's optional `Inst`) vs positional args (the other three builders, 3 required tokens each). Justified by the asymmetry; no struct-for-its-own-sake.

Cuts applied during this review (vs the agent's draft):
- **`MustPromptV2` dropped.** No Slice-2 caller needs the panic-on-construct shape; tests use the error-returning form.
- **`ParsePromptV2` strictness locked.** Each extracted token re-runs `validateToken` (resolves the plan-agent's OQ1; eliminates a leniency asymmetry between build and parse paths).
- **`TestA2A_GetTask_InvalidScope` deferred.** The new failure-path test belongs with the Slice 3+ work that first exposes invalid-scope to a real caller; for Slice 2 the three existing GetTask tests cover the regression surface.
- **`validateToken` stays unexported.** Builder-internal concern; exposing invites validate-then-format split that defeats the builders' point.

External dep count: Slice 2 adds one new module (`github.com/danmestas/sesh-ops` via local `replace`). Transitive cost minor (`oklog/ulid/v2` + a nats.go minor bump).

Proceed to Phase 4.
