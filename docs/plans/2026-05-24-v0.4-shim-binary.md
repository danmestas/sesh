# `sesh-shim` v0.4 — A2A HTTPS+JSON-RPC Gateway Binary

**Goal:** Build `cmd/sesh-shim/`, a stateless Go binary that exposes A2A v1.0 protocol (HTTPS+JSON-RPC + SSE) over an existing sesh mesh. The shim translates A2A on the wire to NATS subjects + direct JetStream KV operations (via the sesh-ops library), enabling any A2A-compliant client (LangChain, CrewAI, custom agents, stock `a2a-python`/`a2a-go` SDK consumers) to address sesh agents.

**Architecture:** Single Go binary in `cmd/sesh-shim/`. Stateless (one signing key on disk). Imports sesh-ops as a Go library (no RPC tier between them). Uses stdlib `net/http` with TLS for HTTPS. Implements the A2A JSON-RPC binding (PascalCase methods: `SendMessage`, `GetTask`, etc.). Connects to NATS as a client; publishes prompts to `agents.prompt.v2.*`; subscribes to `agents.task.stream.*` and KV-watches `sesh_tasks/messages/artifacts_*` buckets to bridge mesh state into SSE for A2A clients.

**Tech Stack:** Go 1.26+, kong CLI framework (matches existing sesh binaries), stdlib `net/http`+TLS, `github.com/nats-io/nats.go` (already in `go.mod` at v1.49.0), `github.com/danmestas/sesh-ops` (NEW dep — public packages after sesh-ops Slice 2). JWS signing via stdlib `crypto/ecdsa` + a small JCS implementation (or pull in `github.com/cyberphone/json-canonicalization` if it suits).

**Brainstorm input:** Design doc at [`../proposals/2026-05-24-synadia-v0.4-a2a-parity-design.md`](../proposals/2026-05-24-synadia-v0.4-a2a-parity-design.md). Audit at [`../proposals/2026-05-24-synadia-v0.4-a2a-parity-audit.md`](../proposals/2026-05-24-synadia-v0.4-a2a-parity-audit.md).

## Architectural decisions specific to this repo

### Package layout

```
cmd/sesh-shim/
  main.go                # kong-based entry; flag/env config; wires together components
internal/shim/           # NEW
  server/                # HTTPS+TLS server, request routing
  jsonrpc/               # JSON-RPC parsing, error envelope, response shaping
  methods/               # one file per A2A RPC method (SendMessage, GetTask, etc.)
  sse/                   # SSE bridging (KV watch + core NATS subscribe → SSE events)
  card/                  # AgentCard L1+L2+L3 composer, JCS canonicalize, JWS sign, in-process LRU cache
  auth/                  # auth middleware (JWT/OAuth2/OIDC/ApiKey), principal extraction, scope mapping
  push/                  # push notification worker (JetStream durable consumer, retry, dead-letter, SSRF guards, credential encryption)
  errors/                # sesh-ops error → A2A error translation
internal/subject/        # NEW (shared with adapters at the protocol-spec level)
  prompt.go              # PromptV1(...), PromptV2(...), ParsePromptV2(...)
  card.go                # CardGet(...), CardExtended(...)
  taskstream.go          # TaskStream(...)
  shutdown.go            # ...etc
```

Why `internal/shim/`: the shim is single-binary; nothing else in sesh consumes it. Internal package avoids cluttering the public API.

Why `internal/subject/`: the v0.4 design uses typed subject builders to avoid string-concat drift between shim and adapters. Adapters re-implement in TypeScript (in sesh-channels); the Go side lives here.

### CLI surface

`cmd/sesh-shim/` is a long-running daemon. Kong is overkill, but matches the rest of the repo and gives consistent help. One command:

```
sesh-shim [--config=path] [--listen=0.0.0.0:8443] [--nats=$NATS_URL] [--dev]
```

Most config lives in `shim.toml` (per design doc Section 7). Flags override.

### Sesh-ops as library import

The shim's go.mod adds:
```
require github.com/danmestas/sesh-ops vX.Y.Z
```

Imports use the public top-level packages after sesh-ops Slice 2 (`task`, `messages`, `artifacts`, `objects`, `scope`, `conn`). The shim does NOT import sesh-ops's CLI command code or its `internal/` (now-empty) tree.

### Stdlib HTTP server + TLS

```go
srv := &http.Server{
    Addr:      cfg.Listen,
    Handler:   shimHandler,
    TLSConfig: &tls.Config{MinVersion: tls.VersionTLS12},
}
srv.ListenAndServeTLS(cfg.TLSCert, cfg.TLSKey)
```

No third-party HTTP framework (gin, echo, chi) — A2A's surface is small; stdlib is sufficient and avoids dependency weight.

### JSON-RPC binding only (REST deferred)

The shim implements ONLY the JSON-RPC binding. REST binding (`POST /message:send`) is out of scope for v0.4 (design doc executive summary). If needed later, REST is a thin shim over the same internal methods.

### Subject-builder package (shared spec source)

`internal/subject/` is the canonical Go source for v0.4 subject patterns. Adapter SDKs (TypeScript) re-implement the same patterns; treat both as implementations of a single protocol spec documented in the design doc. Diverging implementations will surface during integration testing.

## Tracer-bullet slices

Maps to design doc Phases 3 and 5; split into shippable slices.

### Slice 1 — `cmd/sesh-shim/` skeleton (design doc Phase 3)

Files:
- `cmd/sesh-shim/main.go` — kong CLI, env/flag config loading, signal handling, graceful shutdown
- `internal/shim/server/server.go` — `http.Server` setup + TLS + routes mux
- `internal/shim/jsonrpc/envelope.go` — JSON-RPC parse + response wrapping
- `internal/shim/card/composer.go` — L1+L2 composer (L3 deferred to Slice 5)
- `internal/shim/card/signer.go` — JCS canonicalize + JWS sign (ES256)
- `internal/shim/card/cache.go` — in-process LRU (5-min TTL)
- `internal/shim/auth/jwt.go` — Bearer/JWT validation against JWKS
- HTTP endpoints implemented:
  - `GET /.well-known/agent-card.json` — returns L1+L2 composed AgentCard (skeleton)
  - `GET /.well-known/jwks.json` — exposes shim's public signing key(s)
  - `GET /healthz`, `GET /readyz`, `GET /metrics`
  - `POST /a2a` — JSON-RPC entry. Implements TWO methods initially: `GetTask`, `GetExtendedAgentCard`

Acceptance:
- Boots against a dev NATS, fetches AgentCard for a real v0.3 adapter (L2 from `$SRV.INFO.agents`), card JWS-verifies
- `GetTask` returns Task JSON for an existing task ID, `TaskNotFoundError` for bogus ID
- Health/ready/metrics endpoints respond

### Slice 2 — Subject builders + sesh-ops import

Files:
- `internal/subject/` package per layout above
- `cmd/sesh-shim/main.go` wires sesh-ops `conn.Connect(...)` for NATS connection
- `internal/shim/` uses `sesh-ops/task` and `sesh-ops/scope` for Task reads

Acceptance:
- Subject builder unit tests cover round-trip (Build → Parse → equality)
- Shim's `GetTask` from Slice 1 now goes through `sesh-ops/task.Get(...)` instead of inline KV access

### Slice 3 — Send + Stream methods (design doc Phase 5 partial)

Files:
- `internal/shim/methods/send.go` — `SendMessage` (blocking + non-blocking via `returnImmediately` config)
- `internal/shim/methods/stream.go` — `SendStreamingMessage` + SSE handler
- `internal/shim/sse/bridge.go` — combines KV watch (Tasks, Messages, Artifacts) + core NATS subscribe (`agents.task.stream.*`) into A2A SSE event stream
- `internal/shim/sse/dedup.go` — (messageId, artifactId) de-dup
- `internal/shim/methods/subscribe.go` — `SubscribeToTask` reattach
- Idempotency: duplicate-`messageId` check (read existing Message body, compare, return success silently if identical)

Acceptance:
- `SendMessage` end-to-end against a v0.4-capable adapter: writes Message + Task, publishes `agents.prompt.v2.*`, returns Task once terminal
- `SendStreamingMessage` keeps SSE open, emits per-Message and per-Artifact updates as adapter writes them, closes on terminal state
- `SubscribeToTask` reattaches mid-flight, backfills missed Messages from KV history

### Slice 4 — Cancel + List

Files:
- `internal/shim/methods/cancel.go` — `CancelTask` (calls `sesh-ops/task.Cancel`)
- `internal/shim/methods/list.go` — `ListTasks` (filters: status, contextId, created-after/before; cursor-based pagination; authorization filter so principals only see their tasks)

Acceptance:
- `CancelTask` on running task → adapter sees status change, interrupts, SSE closes with `TASK_STATE_CANCELED`
- `CancelTask` on terminal task → `TaskNotCancelableError`
- `ListTasks` returns only tasks the principal can read; pagination cursor round-trips

### Slice 5 — AgentCard L3 + Extended

Files:
- `internal/shim/card/l3.go` — NATS request to `agents.card.get.<agent>.<owner>.<name>`, merge into L1+L2 composition
- `internal/shim/card/extended.go` — NATS request to `agents.card.extended.<agent>.<owner>.<name>`, only called when principal has the `agent.card.extended` scope
- `GetExtendedAgentCard` method completes (currently a stub from Slice 1)
- `sesh mesh card <agent>` subcommand in `cli/mesh.go` (or new `cli/meshcard.go`) — debug helper that returns the composed AgentCard for inspection

Acceptance:
- Adapter that implements both subjects returns a richer extended card; unauthenticated client gets public card; authenticated-with-scope client gets extended
- Adapter that doesn't implement extended → shim returns `AuthenticatedExtendedCardNotConfiguredError`
- `sesh mesh card <agent>` prints the composed card pretty

### Slice 6 — Push notifications

Files:
- `internal/shim/methods/pushcfg.go` — `Create/Get/List/DeleteTaskPushNotificationConfig` methods
- `internal/shim/push/worker.go` — JetStream durable consumer on Tasks KV stream, dispatches webhook POSTs
- `internal/shim/push/ssrf.go` — webhook URL validation: scheme allowlist, DNS resolution + private-IP rejection, optional operator-configured hostname allowlist
- `internal/shim/push/encrypt.go` — `PushConfig.Auth.Credentials` encrypt-at-rest using shim-held key (or KMS)
- `internal/shim/push/retry.go` — exponential backoff, dead-letter to `sesh_notifyfail_*` after exhaustion
- `internal/shim/push/idempotency.go` — adds a delivery-id header so receivers can dedupe

Acceptance:
- Register webhook → status change fires → webhook POST received (with retries on simulated 503)
- Webhook URL with internal IP rejected at registration
- Credentials encrypted in KV (cannot read plaintext via `nats kv get`)
- Failure exhaustion writes to `sesh_notifyfail_*` and logs

### Slice 7 — Object Store URI translation

Files:
- `internal/shim/server/objects.go` — `GET /obj/<scope-kind>/<scope-id>/<task-id>/<artifact-id>` HTTPS download path; auth-gated; reads from `sesh_objects_*` Object Store via `sesh-ops/objects` library
- Translation: A2A `obj://...` URI ↔ HTTPS `https://<shim-host>/obj/...` URL on the way out; reverse on the way in (clients can upload via the same path)
- Optional: presigned/short-lived URLs (TBD by reviewer; non-presigned is acceptable for v0.4 since auth is at the gateway)

Acceptance:
- File written by adapter via `sesh-ops/objects.Put(...)` becomes downloadable via the HTTPS URL
- Cross-scope download attempts rejected by auth
- Round-trip: client uploads via HTTPS, adapter sees `obj://` URI in Message, downloads via Object Store

### Slice 8 — CI workflow

Files:
- `.github/workflows/ci.yml` — Go 1.26, `go test ./...`, `golangci-lint`, integration test gated on `docker` availability
- `.github/workflows/release.yml` — `goreleaser` for the shim binary (TBD: matches existing sesh release patterns; if none exist, propose; ask user)
- Update `cmd/sesh-shim/README.md` if/when created

Acceptance: CI green on PR; release builds reproducibly.

### Slice 9 — End-to-end test with stock A2A client

- Bring up sesh hub + at least one v0.4-capable adapter (sesh-channels Slice 4b output)
- Bring up shim configured to point at the hub
- Run `a2a-python` (or `a2a-go`'s example client) against the shim:
  - Fetch AgentCard
  - Send a message
  - Subscribe to streaming response
  - Cancel a task
  - List tasks
  - Register and trigger a push notification

This is the "you can ship v0.4" gate.

## Rollback plan

Shim is a separate binary. Rollback = stop the process. No external state to clean up (KV buckets persist; the shim's signing key + cache are ephemeral process-local state).

## Open questions for implementation

1. **JCS library choice**: stdlib + small canonicalizer vs. third-party. Recommendation: small inline canonicalizer (JCS is ~100 LOC); avoids dependency.
2. **JSON-RPC library**: implement inline (small surface) vs. pull in third-party. Recommendation: implement inline; library overhead exceeds the surface.
3. **JWKS rotation cadence**: out of scope for code, but the operator playbook needs to exist before production. Document in `cmd/sesh-shim/README.md` once written.
4. **Object Store presigned URLs**: defer to post-v0.4 unless ops needs it (would require time-bounded HMAC over scope+task+artifact).
5. **Multi-tenant deployment**: per design doc Section 7, optional `scope_kind`/`scope_ids` config restricts shim to one scope. Slice ordering doesn't need this; add in Slice 1 if simple, else Slice 6/7.

## Out of scope (deferred)

- REST binding (only JSON-RPC in v0.4)
- gRPC binding
- Service mesh / Envoy sidecar features
- A2A payload schema validation beyond routing requirements
- Multi-protocol gateway
