# v0.4 sesh-shim Slice 4 — CancelTask + ListTasks + Wave D adoption (conn.Connect, task.GetRaw)

**Branch:** `feat/v0.4-shim-slice-4-cancel-list-wave-d`

## 1. Goal

Slice 4 finishes the A2A task-management surface (`CancelTask` + `ListTasks`) and folds in the Wave D sesh-ops adoptions that became available after Slice 3: sesh-ops#24 (`conn.Connect` variadic options) and sesh-ops#25 (`task.GetRaw`). After Slice 4 the shim covers the full v0.4 unary task-API quad (`GetTask`, `SendMessage`, `CancelTask`, `ListTasks`) plus the two streaming methods from Slice 3, and `main.go` boots through the canonical sesh-ops NATS-resolution path with the legacy `nats.JetStreamContext` field dropped from `methods.Deps`. CancelTask honors A2A `TaskNotCancelable` semantics by parsing `status.state` from stored bytes (no full A2A struct round-trip) and CAS-updating in place, preserving any unknown A2A fields. ListTasks ships with a binary auth-scope gate (`agent.read` present → return all; absent → return empty) and no pagination — both per-task ACL granularity and pagination explicitly deferred.

## 2. Success criteria

- `CancelTask` with `params.id` on a non-terminal Task (`TASK_STATE_SUBMITTED|WORKING|INPUT_REQUIRED`) atomically rewrites `status.state = "TASK_STATE_CANCELED"` and returns the updated bytes. KV revision advances by exactly 1.
- `CancelTask` on a terminal Task (`COMPLETED|CANCELED|FAILED|REJECTED|AUTH_REQUIRED`) returns `-32002` with `name == "TaskNotCancelableError"`; KV unchanged.
- `CancelTask` with unknown id or missing bucket returns `-32001 TaskNotFoundError`.
- `CancelTask` with missing/empty `params.id` returns `-32602` with `data.err == "id is required"`.
- `CancelTask` preserves all unknown A2A fields across the cancel (decode-as-map → mutate state → re-encode).
- `CancelTask` retries CAS conflict 3× with backoff (10/20/40ms). On exhaustion: `-32603` with `data.kind == "task-leased"`.
- `ListTasks` with principal carrying `agent.read` returns `{"tasks":[...]}` with every Task in the bucket, byte-passthrough.
- `ListTasks` without `agent.read` returns `{"tasks":[]}` and HTTP 200 (NOT 401, NOT a JSON-RPC error).
- `ListTasks` on empty/missing bucket returns `{"tasks":[]}`.
- `ListTasks` accepts A2A `ListTasksRequest` fields but Slice 4 ignores all except `status` (in-memory exact-match filter). Invalid `status` → `-32602`.
- `ListTasks` response always includes `{tasks, totalSize, pageSize: 0, nextPageToken: ""}` — matches `a2a-go` `ListTasksResponse`.
- Wave D regression: `TestA2A_GetTask_Happy/_NotFound/_BucketMissing` pass byte-identically after `gettask` migrates to `task.GetRaw`.
- Wave D: `methods.Deps.JetStream` (nats.JetStreamContext) removed; `cmd/sesh-shim/main.go` boots via single `conn.Connect(ctx, "hub", cli.NATSURL, "", nats.Name(...), nats.MaxReconnects(-1))` call; `resolveNATSURL` helper deleted.
- `go test ./... -race -count=1` passes; `go vet ./...` and `gofmt -l` clean.

## 3. Non-goals (deferred)

- `SubscribeToCancel` — A2A spec has no such method.
- Per-task ACL filtering in `ListTasks` — Slice 4 ships scope-binary.
- `ListTasks` pagination (`pageSize`/`pageToken` round-trip) — defer until list size matters.
- `ListTasks` filter params other than `status` (`contextId`, `tenant`, `historyLength`, etc.) — accept-but-ignore.
- Cancel propagation to running adapter — Slice 4 mutates KV only; adapter-side wake-up is sesh-channels concern.
- Push notifications (Slice 6), L3 AgentCard (Slice 5), Object Store (Slice 7), CI (Slice 8).
- `task.Cancel` adoption — same Slice 2 D-task rationale: sesh-ops Task struct uses `pending|in_progress|...` status enum, not A2A `TASK_STATE_*`, and lacks A2A fields. Byte-passthrough instead.
- New subject builders — `CancelTask`/`ListTasks` are JSON-RPC only.

## 4. File-by-file design

### 4.1 `internal/shim/methods/cancel.go` (NEW)

```go
package methods

type cancelTaskParams struct {
    ID       string         `json:"id"`
    Metadata map[string]any `json:"metadata,omitempty"`
}

const cancelStatus = "TASK_STATE_CANCELED"

var terminalA2AStates = map[string]struct{}{
    "TASK_STATE_COMPLETED":     {},
    "TASK_STATE_CANCELED":      {},
    "TASK_STATE_FAILED":        {},
    "TASK_STATE_REJECTED":      {},
    "TASK_STATE_AUTH_REQUIRED": {},
}

func (d *Dispatcher) cancelTask(ctx context.Context, params json.RawMessage) (any, *jsonrpc.Error)
func parseStatusState(b []byte) string
func setStatusState(b []byte, state string) ([]byte, error)
```

Body: validate id → `scope.Bucket(..., "tasks")` → open KV → 3-retry loop: `task.GetRaw` → check terminal → `setStatusState` → `kv.Update(..., rev)` → on success return bytes; on CAS conflict backoff + retry. Errors map: `jetstream.ErrBucketNotFound`/`ErrKeyNotFound` → `-32001`; terminal → `-32002`; CAS exhaustion → `-32603 task-leased`.

`parseStatusState` partial-decodes via `struct{ Status struct{ State string } }` (encoding/json drops unknown siblings).

`setStatusState` decodes to `map[string]any`, mutates nested `status.state`, re-encodes. Preserves unknown A2A fields.

### 4.3 `internal/shim/methods/list.go` (NEW)

```go
type listTasksParams struct {
    Tenant, ContextID, Status, PageToken string
    PageSize                              int
    HistoryLength                         *int
    IncludeArtifacts                      bool
}

type listTasksResponse struct {
    Tasks         []json.RawMessage `json:"tasks"`
    TotalSize     int               `json:"totalSize"`
    PageSize      int               `json:"pageSize"`
    NextPageToken string            `json:"nextPageToken"`
}

const readScope = "agent.read"

func (d *Dispatcher) listTasks(ctx context.Context, params json.RawMessage) (any, *jsonrpc.Error)
func hasScope(p auth.Principal, want string) bool
func isKnownA2AState(s string) bool
```

Body: decode params → validate `status` is known A2A state → `auth.FromContext(ctx)` → if no principal or missing `agent.read` → return empty response (NOT 401) → `scope.Bucket(..., "tasks")` → `kv.Keys(ctx)` → for each key: `kv.Get` + status filter → copy bytes into `json.RawMessage`. Always return `{tasks, totalSize, pageSize: 0, nextPageToken: ""}`.

Tasks initialized as `make([]json.RawMessage, 0, ...)` so JSON serializes as `[]` not `null`.
