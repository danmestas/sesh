# v0.4 sesh-shim — Slice 1: skeleton (`cmd/sesh-shim/`)

## 1. Goal

Stand up `cmd/sesh-shim/` as a runnable HTTPS+JSON-RPC daemon that can authenticate (or bypass in dev), serve a signed L1+L2 AgentCard at `/.well-known/agent-card.json`, expose its public JWKS, answer `GetTask` and `GetExtendedAgentCard` over `/a2a`, and pass health/readiness/metrics probes. Unlocks every later slice (Send/Stream, Cancel/List, L3, push, objects) by establishing the routing, error envelope, card composer, signer, and auth chain they all hang off of. After Slice 1, a stock A2A client (e.g. `a2a-go`'s example client) can fetch a card and call `GetTask` against the shim, validating the wire shape end-to-end.

## 2. Success criteria

Acceptance from `docs/plans/2026-05-24-v0.4-shim-binary.md` (verbatim):
- Boots against a dev NATS, fetches AgentCard for a real v0.3 adapter (L2 from `$SRV.INFO.agents`), card JWS-verifies.
- `GetTask` returns Task JSON for an existing task ID, `TaskNotFoundError` for bogus ID.
- Health/ready/metrics endpoints respond.

Sub-criteria (verifiable locally):
- `go build ./cmd/sesh-shim` succeeds on Go 1.26.0.
- `go test ./...` passes; new packages have meaningful coverage on pure code paths (card/signer, jsonrpc/envelope, card/canonicalize).
- `go vet ./...` clean.
- `gofmt -l ./cmd/sesh-shim ./internal/shim` prints nothing.
- `golangci-lint run ./...` clean (if installed; not required by CI yet — Slice 8).
- `sesh-shim --dev --listen=127.0.0.1:18443 --nats=$NATS_URL` boots, traps SIGINT, drains NATS before exiting (mirrors `internal/refagent/agent.go`).
- `curl -sk https://127.0.0.1:18443/healthz` → 200 with JSON `{"status":"ok","nats":<bool>,"signing_key":<bool>}` (booleans, not strings — chosen for readability and consistent JSON typing across probes).
- `curl -sk https://127.0.0.1:18443/.well-known/agent-card.json` returns `application/json` with `signatures[0].signature` populated.
- A round-trip test (Go test) signs a card with the dev signer, fetches it back, JCS-canonicalizes the card-minus-signatures, recomputes the JWS signing input, and verifies the signature against the dev pub key.
- JSON-RPC error path tests (`-32700`/`-32600`/`-32601`/`-32602`/`-32001`/`-32007`) all return the right `code` AND `name`.
- Process drains NATS on SIGINT within 5s.

### 4.1 `cmd/sesh-shim/main.go`

Kong CLI entry. Single top-level struct, no subcommands.

```go
package main

type CLI struct {
    Listen      string        `name:"listen"      default:"0.0.0.0:8443" env:"SESH_SHIM_LISTEN"   help:"HTTPS bind address"`
    NATSURL     string        `name:"nats"        env:"NATS_URL"                                   help:"NATS URL; falls back to ~/.sesh/hub.url"`
    TLSCert     string        `name:"tls-cert"    env:"SESH_SHIM_TLS_CERT"                         help:"PEM TLS certificate; in --dev mode self-signed if empty"`
    TLSKey      string        `name:"tls-key"     env:"SESH_SHIM_TLS_KEY"                          help:"PEM TLS key; in --dev mode self-signed if empty"`
    SigningKey  string        `name:"signing-key" env:"SESH_SHIM_SIGNING_KEY"                      help:"PEM ES256 private key for AgentCard JWS; in --dev mode auto-generated if empty"`
    KeyID       string        `name:"kid"         env:"SESH_SHIM_KID"                              help:"Key ID published in JWKS; auto-derived if empty"`
    Auth        string        `name:"auth"        default:"jwt" enum:"jwt,none-dev-only" env:"SESH_SHIM_AUTH" help:"Auth mode for /a2a"`
    JWKSURL     string        `name:"jwks-url"    env:"SESH_SHIM_JWKS_URL"                         help:"Upstream JWKS URL for JWT validation; required when --auth=jwt"`
    Agent       string        `name:"agent"       env:"SESH_SHIM_AGENT"                            help:"Adapter agent token to advertise (single-agent per shim, see Decision D1)"`
    Owner       string        `name:"owner"       env:"SESH_SHIM_OWNER"                            help:"Adapter owner token"`
    ScopeKind   string        `name:"scope-kind"  env:"SESH_SHIM_SCOPE_KIND" default:"session"     help:"Scope kind for KV bucket lookup"`
    ScopeID     string        `name:"scope-id"    env:"SESH_SHIM_SCOPE_ID"                         help:"Scope id for KV bucket lookup; required for GetTask"`
    Dev         bool          `name:"dev"         help:"Dev mode: self-signed TLS, auto-generated signing key, looser logging"`
    ShutdownGrace time.Duration `name:"shutdown-grace" default:"5s" help:"Max drain/shutdown wait"`
}

func main() // signal.NotifyContext(SIGINT,SIGTERM), kong.Parse with BindTo(ctx), construct deps, invoke server.Run(ctx, cfg).
```

Responsibilities:
- Parse flags via `kong.Parse(&c, kong.Name("sesh-shim"), kong.BindTo(ctx, (*context.Context)(nil)))`.
- Resolve NATS URL (flag → env → `~/.sesh/hub.url`). The shim is process-level rather than session-bound, so the session-file tier from `internal/refagent/agent.go`'s three-tier walk is intentionally omitted.
- Dial NATS: `nats.Connect(url, nats.Name("sesh-shim"), nats.MaxReconnects(-1))`.
- Construct dependencies in order: signer → card composer (needs NATS) → cache (wraps composer+signer) → auth validator → server.
- Block on `ctx.Done()` then call `server.Shutdown(grace-ctx)` then `nc.Drain()`.
- Exit non-zero on startup error; log via `slog.NewJSONHandler` to stdout.

Does NOT: load TOML, instantiate push worker, instantiate SSE bridge, import sesh-ops.

### 4.2 `internal/shim/server/server.go`

```go
package server

type Config struct {
    Listen      string
    TLSCert     string // empty + Dev=true → self-signed
    TLSKey      string
    Dev         bool
    Auth        auth.Validator        // injected; may be auth.NoopValidator{} in dev
    Card        *card.Cache           // signed-card cache (cache wraps composer+signer)
    Signer      *card.Signer          // for /.well-known/jwks.json
    NC          *nats.Conn            // for healthz NATS check + Slice 1 inline KV reads
    JetStream   nats.JetStreamContext // for GetTask
    AgentKey    card.AgentKey         // single-agent per shim in Slice 1 (D1)
    ScopeKind   string
    ScopeID     string
    Logger      *slog.Logger
    ShutdownGrace time.Duration
}

// New constructs the *http.Server with all routes mounted.
func New(cfg Config) (*http.Server, error)

// Run is the convenience entry main.go uses: starts ListenAndServeTLS
// (or the dev self-signed variant) and blocks until ctx is cancelled,
// then calls Shutdown with cfg.ShutdownGrace.
func Run(ctx context.Context, cfg Config) error
```

Routes mounted on a single `http.ServeMux`:
- `GET /.well-known/agent-card.json` → handler calls `cache.GetOrCompose(ctx, cfg.AgentKey)` → write bytes.
- `GET /.well-known/jwks.json` → handler calls `signer.JWKS()` → write JSON.
- `GET /healthz` → JSON `{status, nats, signing_key}` based on `nc.Status()` and `signer.Loaded()`.
- `GET /readyz` → 200 iff NATS connected (Slice 1; see Open Question 3 for the "adapter-present" deferral).
- `GET /metrics` → Prometheus exposition format. Slice 1 ships counters: `sesh_shim_http_requests_total{method,route,status}`, `sesh_shim_jsonrpc_errors_total{code,name}`, `sesh_shim_card_cache_hits_total`, `sesh_shim_card_cache_misses_total`. Inline ~60 LOC text emission; do NOT pull in `prometheus/client_golang` for Slice 1.
- `POST /a2a` → auth middleware → `jsonrpc.Decode` → method dispatch → `jsonrpc.WriteResponse` or `jsonrpc.WriteError`.

TLS:
- If `cfg.TLSCert != ""` and `cfg.TLSKey != ""` → `srv.ListenAndServeTLS(cert, key)` with `TLSConfig{MinVersion: tls.VersionTLS12}`.
- If both empty AND `cfg.Dev=true` → generate in-memory ES256 self-signed cert via `devtls.go`, set `srv.TLSConfig.Certificates`, call `srv.ListenAndServeTLS("", "")`.
- If both empty AND `cfg.Dev=false` → return error at New().

Does NOT: SSE, websocket, REST binding, rate limiting.

Houses dispatch + the two Slice-1 method implementations. Same-package keeps Slice 1 simple; promote to `internal/shim/methods/` in Slice 3 when more methods land (per design Section 7 layout).

```go
// getTask handles JSON-RPC method "GetTask". Slice 1 reads the Task

// key=<task-id>. Slice 2 moves this through sesh-ops/task.Get(...).
```

`getTask` flow:
1. Parse params: `{"id": "<task-id>", "historyLength": <int>}`. `historyLength` ignored in Slice 1.

### 4.4 `internal/shim/card/composer.go`

```go
package card

// Composer composes L1 (shim defaults) + L2 (adapter $SRV.INFO fallback)
// into a pristine *a2a.AgentCard. Per audit P1.4, Slice 1 implements ONLY
// the $SRV.INFO fallback path — the rich `agents.card.get.*` fetch lands
// in Slice 5.
type Composer struct {
    nc          *nats.Conn
    l1          L1Defaults
    queryWindow time.Duration
}

type L1Defaults struct {
    GatewayURL         string
    ProtocolVersion    string                // typically "1.0"
    Provider           *a2a.AgentProvider
    SecuritySchemes    map[string]any
    Capabilities       a2a.AgentCapabilities // streaming=false, pushNotifications=false in Slice 1
    DefaultInputModes  []string              // []string{"text/plain"}
    DefaultOutputModes []string              // []string{"text/plain"}
}

type AgentKey struct {
    Agent string
    Owner string
}

func NewComposer(nc *nats.Conn, l1 L1Defaults, queryWindow time.Duration) *Composer
func (c *Composer) Compose(ctx context.Context, key AgentKey) (*a2a.AgentCard, error)
```

L2 merge contract (Slice 1):
- `name` ← `info.Metadata["agent"]`
- `version` ← `info.Metadata["harness_ver"]` if present, else `"0.0.0"`
- `defaultInputModes` ← L1
- `defaultOutputModes` ← L1
- `description` ← derived from `info.Metadata["role"] + "/" + info.Metadata["class"]` if present, else empty
- everything else from L1

`supportedInterfaces` (audit P0.5 — required) in L1 is one entry:
```go
[]*a2a.AgentInterface{a2a.NewAgentInterface(l1.GatewayURL, a2a.TransportProtocolJSONRPC)}
```

Behavior on no `$SRV.INFO` match: return the L1 skeleton (always well-formed) and log WARN. Cache for the standard TTL — operator restarts the shim if it boots before the adapter (acceptable for Slice 1).

Does NOT: call `agents.card.get.*` (Slice 5); call `agents.card.extended.*` (Slice 5); sign (signer); cache (cache).

**Vendor decision**: import `github.com/a2aproject/a2a-go/v2/a2a` for typed struct shapes. Single dep; JSON tags are authoritative.

### 4.5 `internal/shim/card/signer.go`

```go
package card

type Signer struct {
    priv  *ecdsa.PrivateKey
    pub   *ecdsa.PublicKey
    kid   string
}

func LoadSigner(path, kid string) (*Signer, error)
func NewDevSigner() (*Signer, error)         // ephemeral P-256 keypair; logs kid at INFO
func (s *Signer) Loaded() bool
func (s *Signer) JWKS() ([]byte, error)
func (s *Signer) SignCard(card *a2a.AgentCard) ([]byte, error)
```

Algorithm in `SignCard`:
1. Deep-copy the card; set `card.Signatures = nil`.
2. JCS-canonicalize via `canonicalize.go`.
3. Build protected header bytes: JSON `{"alg":"ES256","kid":"<kid>","typ":"JWT"}` → base64url no padding.
4. Signing input = `protected_b64 || "." || base64url(canonicalized_card_bytes)` per JWS RFC 7515 §5.1.
5. Hash SHA-256, ECDSA-sign, convert ASN.1 (R, S) → fixed 32+32-byte concat (R || S), base64url no padding (RFC 7518 ES256).
6. Set `card.Signatures = []a2a.AgentCardSignature{{Protected: prot_b64, Signature: sig_b64}}` and marshal the card for the response body.

#### 4.5.1 `internal/shim/card/canonicalize.go`

Inline RFC 8785 JCS implementation, stdlib only, ~100–150 LOC (audit-mandated; no third-party dep). Algorithm:
- `json.Unmarshal` into `any`.
- Recursive walk:
  - Object → sort keys lexicographically by UTF-16 code units (RFC 8785 §3.2.3); emit `{"k":v,…}` no whitespace.
  - Array → `[v,…]` no whitespace, preserve order.
  - String → JCS string form (escape `"`, `\`, control chars only; everything else literal UTF-8). Use `json.Encoder` with `SetEscapeHTML(false)`, strip trailing newline.
  - Number → JCS number form via `strconv.FormatFloat(v, 'g', -1, 64)` with post-processing for shortest unique decimal; `-0` → `0`.
  - Bool/null → as-is.

```go
func canonicalizeJSON(input []byte) ([]byte, error)
```

Test surface (paramount — JCS bugs are silent verifier failures):
- RFC 8785 §3.2 worked examples as table-driven tests.
- Round-trip: random JSON → canonicalize → parse → canonicalize → bytes-equal.
- Ordering: `{"b":1,"a":2}` → `{"a":2,"b":1}`.
- Number edge: `{"x": 1.0}` → `{"x":1}`.

### 4.7 `internal/shim/auth/jwt.go`

```go
package auth

type Validator interface {
    Validate(r *http.Request) (Principal, error)
}

type Principal struct {
    Sub    string
    Scopes []string
}

func Middleware(v Validator) func(http.Handler) http.Handler
func FromContext(ctx context.Context) (Principal, bool)

type NoopValidator struct{ Logger *slog.Logger }

type JWTValidator struct {
    jwksURL  string
    cache    *jwksCache
    httpc    *http.Client
    Audience string
    Issuer   string
}

func NewJWTValidator(jwksURL string) (*JWTValidator, error)
```

`Validate` for `JWTValidator`:
1. Extract `Authorization: Bearer <token>` (RFC 6750). Missing → 401.
2. Parse + verify with `github.com/golang-jwt/jwt/v5`, key function = JWKS lookup by `kid`.
3. Verify `exp`/`nbf`/`iat`. Optional `iss`/`aud`.
4. Extract `sub`, `scope` (split space-separated).
5. Return Principal.

JWKS cache: in-process map keyed by `kid`, refreshed via HTTP GET against `jwksURL`. ~60 LOC hand-parse for `RS256`/`ES256` only. No `go-jose` dep.

JWKS-fetch failure → 503 (not 401) — distinguish "no cached key, upstream down" from "bad token".

`NoopValidator`: returns `Principal{Sub:"dev", Scopes:["agent.read","agent.write"]}` for every request. Logs WARN per accepted request so production misconfigs are loud.

Does NOT: OAuth2 flows, OIDC discovery, ApiKey schemes.

## 5. Decisions resolved

| # | Question | Decision | Reason |
|---|---|---|---|
| D1 | Multi-agent vs single-agent in Slice 1 | Single agent per shim instance via `--agent`/`--owner`/`--scope-id` flags; `/.well-known/agent-card.json` serves that one card | Satisfies acceptance with simplest model; multi-agent routing is a Slice 5+ concern |
| D2 | Task storage access | Inline JetStream KV reads at bucket `sesh_tasks_<scope-kind>_<scope-id>`; Slice 2 swaps for `sesh-ops/task.Get(...)` | Avoids blocking Slice 1 on sesh-ops Slice 2 timing |
| D3 | JWT auth scope | `--auth=jwt\|none-dev-only` enum matching design Section 7 `[auth].mode`; `none-dev-only` bypasses with WARN-logged placeholder principal | Matches design's enum verbatim; dev acceptance without IDP setup |
| D4 | TLS in dev | On-the-fly self-signed ES256 cert when `--dev` AND both `--tls-cert`/`--tls-key` empty; `internal/shim/server/devtls.go`, stdlib only | No external dep; operator can override |
| D5 | Signing key in dev | `--dev` auto-generates ES256 keypair in-memory, logs `kid` at INFO; production REQUIRES `--signing-key=<path>` to a PEM | Avoids first-boot mkdir prompts; rotation is an ops concern |
| D-config | TOML vs flags+env | Flags + env only in Slice 1; defer `BurntSushi/toml` until a slice needs it | sesh `go.mod` has no TOML lib; reversible |
| D-deps | A2A struct shapes | `github.com/a2aproject/a2a-go/v2/a2a` | Single audited dep; struct shapes + JSON tags authoritative (audit P0.5/P0.6 baked in) |
| D-jwt | JWT library | `github.com/golang-jwt/jwt/v5` | Canonical, widely audited, single dep |

## 7. Risks

| Risk | Mitigation |
|---|---|
| JCS edge-case bugs cause silent JWS verification failures | Anchor tests on the four RFC 8785 §3.2 vectors verbatim; round-trip idempotence test. |
| ES256 signature encoding mismatch (ASN.1 DER vs R\|\|S concat) | Cross-verify with hand-decoded R\|\|S split + `ecdsa.Verify`. |
| `$SRV.INFO` window race — composer fires before adapter ready → cached L1-only skeleton for 5 min | Documented; operator restart fixes. Slice 5 cache invalidation removes the workaround. |
| JWKS HTTP failure during JWT verify → 401 storm | Validator returns 503 on JWKS-fetch failure, not 401. |
| Self-signed dev TLS rejected by strict A2A clients | Document curl `-k`; suggest `mkcert` for higher-fidelity dev. Production refuses empty TLSCert+TLSKey when Dev=false. |
| Single-agent-per-shim (D1) leaks into types that later need multi-agent | `AgentKey{Agent, Owner}` is already composite; Slice 5 only changes how it's *resolved* per request. |

## Phase 3 — Ousterhout backpressure review

Applied to the plan as it stands:

(a) **Shallow modules.** `NoopValidator` (~10 LOC) is shallow on purpose — it satisfies the same `Validator` interface as `JWTValidator` and removes an `if auth == "none" { ... } else { ... }` branch at every handler. Earns its keep. `devtls.go` (~60 LOC) is also small but its one function hides nontrivial x509/ECDSA boilerplate that would clutter `server.go`; accept until a second caller appears (don't pre-promote to `internal/dev/tlscert`).

(b) **Pass-through wrappers.** `server.Run` wraps `server.New` + `ListenAndServeTLS` + Shutdown — but it absorbs dev-TLS vs prod-TLS branching and graceful-shutdown plumbing, keeping `main.go` short. Not a pure pass-through; it earns its keep.

(c) **Over-generalization.** `AgentKey{Agent, Owner}` could be a single string in Slice 1 (single agent per shim per D1). Kept composite because Slice 5 needs multi-agent and a Slice-5 change to widen the Cache/Composer keys is more disruptive than holding the composite shape now. One field of foresight, not three.

(d) **JCS as inline ~100 LOC, no dep.** Confirmed. `canonicalize.go` is stdlib only; signer imports only `crypto/*` stdlib + the project's chosen JWT lib (`golang-jwt/jwt/v5`).

(e) **Cut from the original plan after review:**
- `cmd/sesh-shim/devtools/verify-card/main.go` — out of Slice 1 scope per the mission's file list. The plan-agent suggested it as a convenience; the round-trip is covered by `signer_test.go` and manual acceptance can verify via shell tools.
- "First-fetch path that does NOT cache empty/skeleton" — over-engineering for Slice 1. Single behavior (cache for TTL; restart shim if it raced the adapter) is simpler and the documented mitigation matches.

(f) **Module count check.** Slice 1 adds five Go packages (`jsonrpc`, `card`, `auth`, `server`, `cmd/sesh-shim`). Each has a focused responsibility and a clean public surface. Card stays one package (composer + signer + canonicalize + cache) because they share the typed card model and the signer needs the canonicalizer. Splitting `card` into 3-4 packages would create deep import chains for no benefit.

(g) **External dep check.** Slice 1 adds two third-party imports total: `github.com/a2aproject/a2a-go/v2/a2a` (struct shapes) and `github.com/golang-jwt/jwt/v5` (token parsing). Both are audited, single-purpose, widely deployed. Acceptable.

No structural rework required. Proceed to Phase 4.
